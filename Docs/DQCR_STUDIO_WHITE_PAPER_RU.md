# DQCR Studio White Paper

## Кратко о продукте

**DQCR Studio** - это специализированная workspace-платформа для разработки, сопровождения, проверки и сборки DQCR-проектов. Продукт объединяет в одном интерфейсе файловую структуру проекта, редакторы SQL и YAML, визуальное редактирование моделей, анализ зависимостей, управление параметрами, продвинутый autocomplete, поддержку внешнего каталога данных, валидацию и сборку артефактов.

Ключевая идея продукта: дать команде единое рабочее пространство для полного жизненного цикла DQCR-проекта без разрыва между файлами, framework CLI и пользовательским интерфейсом.

## Для кого продукт

DQCR Studio ориентирован на команды, которые разрабатывают и поддерживают DQCR-проекты в промышленной среде.

Основные пользователи:

- **Data Engineer / Analytics Engineer** - пишет и сопровождает SQL, параметры, модели и workflow.
- **DQCR-архитектор / техлид** - контролирует структуру проектов, правила, шаблоны и качество решений.
- **Аналитик / power user** - вносит изменения в модели и конфигурацию через low-code/no-code сценарии.
- **Администратор платформы** - управляет каталогами, шаблонами, validation rules и служебной конфигурацией.

## Какую задачу решает DQCR Studio

Во многих командах DQCR-проект живет сразу в нескольких плоскостях:

- как набор файлов в папках;
- как набор соглашений, которые знает только опытная команда;
- как процесс validate/build, который запускается отдельными CLI-командами;
- как бизнес-логика, которую трудно быстро просмотреть и безопасно изменить.

Из-за этого возникают типовые проблемы:

- долгое онбординг-погружение в структуру проекта;
- высокая цена ошибки при ручном редактировании YAML и SQL;
- сложность навигации по моделям, контекстам, параметрам и workflow;
- разрыв между редактированием, проверкой и сборкой;
- низкая прозрачность зависимостей и происхождения итогового SQL/артефактов.

**DQCR Studio решает эту проблему**, превращая DQCR-проект из "набора файлов и команд" в управляемую инженерную среду с прозрачной структурой, встроенными проверками и единым UX для всей команды.

## Что умеет продукт

DQCR Studio покрывает ключевые рабочие сценарии команды:

- управление портфелем проектов через **Project Hub**;
- открытие проекта в **Workbench** с файловым деревом и IDE-подобной навигацией;
- создание и редактирование моделей, файлов, папок, параметров и контекстов;
- визуальное и YAML-редактирование `model.yml`;
- редактирование SQL с Monaco Editor;
- продвинутый autocomplete с учетом структуры проекта, моделей, workflow-объектов, параметров и сущностей каталога;
- просмотр зависимостей модели в **Lineage**;
- запуск **Validate** с отображением ошибок, предупреждений и quick fixes;
- запуск **Build** для генерации целевых артефактов;
- работу с глобальными и модельными параметрами;
- подключение, загрузку и использование внешнего каталога данных;
- поиск сущностей каталога и импорт атрибутов в модели;
- администрирование шаблонов, макросов и правил;
- использование встроенного terminal/logs/output внутри Studio.

## 3 Killer Features

### 1. Filesystem-first подход без потери удобства IDE

DQCR Studio не прячет проект за абстрактной внутренней моделью. Источником истины остается реальная файловая структура проекта: `project.yml`, `contexts`, `parameters`, `model`, SQL и workflow-файлы. Это снижает vendor lock-in, упрощает интеграцию с Git и делает продукт понятным для инженерной команды.

При этом пользователь получает полноценный IDE-like интерфейс поверх файлов: дерево проекта, редакторы, навигацию, операции с файлами и моделями, не теряя прозрачности структуры.

### 2. Единая среда для редактирования, анализа, проверки и сборки

Обычно работа с DQCR-проектом разбита между редактором кода, терминалом, отдельными скриптами и ручной диагностикой. В DQCR Studio эти этапы собраны в одном продукте:

- открыл проект;
- изменил модель или SQL;
- проверил зависимости;
- запустил validate;
- применил quick fix;
- собрал итоговый результат.

Это заметно сокращает время цикла "изменил -> проверил -> выпустил".

### 3. Прозрачность сложной логики проекта

DQCR Studio делает сложную структуру DQCR-проекта обозримой:

- показывает lineage и workflow-зависимости;
- помогает понять параметры, контексты и generated output;
- дает object-aware autocomplete по проекту и каталогу данных;
- связывает модели с внешним каталогом данных и его сущностями;
- дает доступ к validation и build-артефактам;
- связывает файловую структуру с вычисляемыми представлениями framework.

За счет этого продукт снижает зависимость от "знаний в голове эксперта" и делает проект управляемым для всей команды, а не только для 1-2 ключевых специалистов.

## Бизнес-ценность

Для команды и организации DQCR Studio дает следующие эффекты:

- ускорение разработки и сопровождения DQCR-проектов;
- снижение количества ошибок при изменении SQL, YAML и параметров;
- ускорение онбординга новых специалистов;
- повышение прозрачности и воспроизводимости validate/build процессов;
- единый стандарт работы с проектами, моделями и правилами;
- сокращение времени на диагностику проблем и выпуск изменений.

## Итог

DQCR Studio - это не просто редактор файлов и не просто frontend над framework CLI. Это специализированная инженерная среда для команд, которые хотят управлять DQCR-проектами быстрее, безопаснее и прозрачнее.

Продукт особенно ценен там, где важны:

- сложная файловая структура проекта;
- большое количество SQL и модельных артефактов;
- необходимость постоянной валидации и сборки;
- совместная работа нескольких ролей в одном процессе.
