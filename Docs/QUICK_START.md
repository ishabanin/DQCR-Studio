# DQCR Studio — Быстрый старт

**Дата:** 22 марта 2026

## 1. Что нужно заранее

- Docker + Docker Compose
- GNU Make
- свободный порт `80` (nginx), `5173` (frontend), `8000` (backend)

## 2. Запуск за 2 минуты

1. Перейдите в корень репозитория.
2. Выполните:

```bash
make dev
```

3. Откройте в браузере:

- `http://localhost:80`

4. Проверьте health:

- `http://localhost:80/health`
- `http://localhost:80/ready`

Если ответы `ok/ready`, Studio готова к работе.

## 3. Первый рабочий цикл (5–10 минут)

1. В `All projects` выберите проект `sample` (или создайте новый через `+ New project`).
2. Перейдите на вкладку `Project info` и проверьте, что видите модели/контексты/параметры.
3. Откройте `Model editor`, выберите модель и внесите небольшое изменение (например, описание).
4. Откройте SQL-файл в `SQL editor`, измените текст и сохраните (`Ctrl/Cmd + S`).
5. Нажмите `Validate` в верхней панели.
6. Перейдите на вкладку `Build` и запустите сборку с `engine = dqcr`, `context = default`.
7. Откройте сгенерированный файл в дереве Output, при необходимости скачайте ZIP.

## 4. Где что находится

- Проекты в файловой системе: `projects/`
- Каталог сущностей: `catalog/`
- Системная документация: `Docs/SYSTEM_REFERENCE.md`
- Подробная пользовательская документация: `Docs/USER_GUIDE.md`

## 5. Полезные команды

```bash
make dev      # запуск окружения
make down     # остановка контейнеров
make test     # backend + frontend тесты
```

## 6. Если что-то не работает

- Проверьте, что порты `80/5173/8000` не заняты.
- Если UI не открывается, посмотрите логи контейнеров:

```bash
docker compose -f infra/docker/docker-compose.yml logs -f
```

- Если `Validate`/`Build` не выполняются, проверьте `http://localhost:80/ready` и наличие выбранной модели.
