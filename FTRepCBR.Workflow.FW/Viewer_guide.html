1. Введение

FW Workflow Viewer — веб-приложение для визуализации и анализа проектов DQCR Framework. Viewer позволяет загружать проекты, просматривать структуру, анализировать SQL-запросы, строить граф зависимостей и запускать валидацию.

Архитектура:
Frontend: React (порт 3000) + Backend: FastAPI (порт 9001)

4.3 Контекстные бейджи

На элементах дерева могут отображаться контекстные бейджи — зеленые метки, показывающие к каким контекстам относится файл:

📄 001_Query_vtb.sql
vtb
📁 002_Update
default vtb

Стиль контекстных бейджей:

default vtb all

Когда отображаются:

  • Бейдж показывается, если шаг/файл относится к конкретному контексту
  • Если файл относится ко всем контекстам — бейдж не показывается (или показывается "all")
  • Бейджи помогают понять, какие файлы будут использоваться в каком контексте

4.4 Навигация

  • Клик по элементу — открыть файл в новой вкладке
  • Клик по папке — раскрыть/скрыть содержимое
  • Перетаскивание границы — изменить ширину sidebar
  • Выбор элемента — подсветка синим цветом

6.1 Режимы просмотра SQL

Режим Описание
Source Оригинальный SQL из файла
Prepared SQL после замены параметров, но до материализации
Rendered Финальный SQL с материализацией

6.2 Выбор инструмента (tool)

Для режимов Prepared и Rendered доступен выбор инструмента:

  • adb — Apache Doris / ADB
  • oracle — Oracle
  • postgresql — PostgreSQL

9. Валидация

Нажмите Verify в toolbar для запуска валидации. После выполнения валидации:

  1. В дереве проекта — индикаторы (точки) на файлах и папках
  2. При клике на файл — детальная информация об ошибках

Типы индикаторов:

Логика отображения:

  • Индикатор показывается на файле, где найдена проблема
  • Индикатор также показывается на родительских папках (суммируются)
  • Если файл имеет ошибку и предупреждение — показывается только ошибка (высший приоритет)
  • Самый высокий уровень проблемы в папке показывается на самой папке

10. Запуск

10.1 Требования

  • Python 3.10+
  • Node.js 18+
  • npm

10.2 Запуск Viewer

python -m FW.viewer.run_viewer

После запуска:

  • Frontend: http://localhost:3000
  • Backend: http://localhost:9001
  • API Docs: http://localhost:9001/docs
Важно: Скрипт автоматически останавливает процессы на портах 3000 и 9001 перед запуском.

Приложение А. Устранение неполадок

А.1 Viewer не запускается

Решение:
  1. Проверьте Node.js: node --version
  2. Установите зависимости: cd FW/viewer/frontend && npm install

А.2 Порты заняты

Решение:
taskkill /F /IM node.exe

А.3 Проект не загружается

Решение:
  1. Проверьте путь к проекту
  2. Проверьте наличие project.yml
  3. Проверьте структуру директорий
